## TL;DR

Whilly Orchestrator — Python-обвязка вокруг Ralph-loop техники Ghuntley:
бери задачу → агент → verify → коммит. Поверх классического loop'а добавили
**TRIZ-анализ противоречий, Decision Gate** (отказ от мусорных задач до LLM-расхода)
и **PRD-визард**. За 6 дней уехало **6 релизов в PyPI** (v3.0 → v3.3),
**~12k строк Python**, **661 тест** на Linux/macOS/Windows × Python 3.10/3.11/3.12.

Курс vNext — **Whilly Forge: Issue → Draft PR за один прогон.**

## 1. Что планировали и куда движемся

### Базовая идея
Continuous agent loop с предсказуемым ремонтом:
- Источник задач (GitHub Issues / Projects v2 / Jira / `tasks.json` / PRD-визард).
- Pre-flight: TRIZ + Decision Gate + Decomposer.
- Оркестратор бьёт задачи на батчи по непересекающимся `key_files`,
  диспатчит в tmux/worktree, собирает результат через `<promise>COMPLETE</promise>`.
- Verify → Repair (self-heal) → Compose PR → Board sync.

### vNext — Whilly Forge

```
Issue ──► Intake ──► Normalize ──► Readiness ──► Strategy ──► Plan ──► Execute ──► Verify ──► Repair ──► Compose PR ──► Timeline
```

11 ADR/FR-задач (FR-1 … FR-11) разбивают этот pipeline на сменяемые модули:
любая команда может подменить strategy/verifier/PR composer под свой стек.

### Ключевые компоненты пакета `whilly/`

| Слой | Модули |
|---|---|
| Цикл и оркестрация | `cli.py`, `orchestrator.py`, `task_manager.py`, `state_store.py`, `decomposer.py` |
| Бэкенды агентов (Protocol) | `agents/{claude,opencode,claude_handoff}.py` |
| Параллельный запуск | `tmux_runner.py`, `worktree_runner.py` |
| Источники задач | `sources/{github_issues,github_issues_and_project,jira}.py` |
| Sinks (PR/борды) | `sinks/github_pr.py`, `workflow/{github,analyzer,mapper,sync}.py` |
| Преаналитика | `triz_analyzer.py`, `decision_gate.py`, `prd_{wizard,launcher,generator}.py` |
| Надёжность | `self_healing.py`, `recovery.py`, `doctor.py`, `verifier.py` |
| UX | `dashboard.py` (Rich TUI), `reporter.py`, `notifications.py`, `web_status.py` |

  через `.whilly/handoff/<task_id>/prompt.md`, новые статусы `blocked` / `human_loop`
  с отдельными колонками на доске.
- **Cross-platform CI:** Ubuntu × macOS × Windows × Python 3.10/3.11/3.12 —
  9 матриц на каждый PR.
- **Тесты:** 490 → **643** (+153 за релиз).
- **Docs:** GitHub Pages (Jekyll + just-the-docs), Getting-Started walkthrough,
  расширенный `whilly.example.toml` в man-page стиле.

## 4. Главные сложности

1. **Subprocess-spawn под нагрузкой (macOS).**
   Параллельные агенты упирались в `EAGAIN` / `RLIMIT_NPROC` —
   потребовался retry-loop при спавне. На macOS дефолтный лимит процессов
   на пользователя пробивается раньше, чем `ulimit -n`. Fix: [#213](https://github.com/mshegolev/whilly-orchestrator/pull/213).

2. **State source-of-truth.**
   Конкуренция между `tasks.json` на диске и in-memory state — только дисциплиной
   «`mark_status` → `save` → `reload`» + atomic-write. Стэйл `in_progress`
   после краша → `recovery.py` + сброс на старте.

3. **Ghost plans.**
   `.whilly_state.json` мог держать ссылки на уже удалённые планы.
   Решено детектором в `whilly doctor`, а не «починкой по факту».
   Урок: «инвалидация ссылок на внешние артефакты — это feature, а не побочный эффект».
   PR: [#209](https://github.com/mshegolev/whilly-orchestrator/pull/209).

4. **Workspace-by-default оказался анти-фичей.**
   Plan-level git worktree выглядел безопасным, но в пилотах с uncommitted changes
   и абсолютными путями в `.venv` ломал больше, чем защищал. Откатили дефолт
   в v3.3.0 (BREAKING) — это было дорогое, но честное решение.
   PR: [#212](https://github.com/mshegolev/whilly-orchestrator/pull/212).

5. **Парсинг внешних CLI.**
   `gh pr create` иногда печатает `Warning:` строкой stdout-а;
   парсер ловил всё разом и закрывал чистый PR.
   Ушли на строгий regex по PR-URL + отдельный `gh pr merge` без `--delete-branch`.
   Привычка: «никаких whole-stdout-capture для CLI-вывода».

6. **GitHub Projects v2 пагинация.**
   Доски с >100 карточек получали HTTP 400 на каждой live-sync транзакции.
   `items(first: 100)` без пагинации — баг, который проявляется только в проде.

7. **Полиплатформенный лог.**
   Cyrillic-преамбула падала на Windows cp1252, `termios`/`tty` нет на Windows.
   Поправили UTF-8 + guard на импортах. После этого CI на 3 OS стал зелёным.

8. **Token shadowing.**
   `GITHUB_TOKEN` в env молча перебивал keyring-аутентификацию `gh` в 7 местах.
   Централизовали разрешение через `WHILLY_GH_TOKEN` / `WHILLY_GH_PREFER_KEYRING`.
   Memory-rule: «`unset GITHUB_TOKEN` перед `gh` командами».

9. **`<promise>COMPLETE</promise>` как контракт.**
   Без жёсткого маркера завершения агенты «зацикливались, изображая работу».
   Решение — единый промпт-билдер и проверка `is_complete` в `agent_runner`.
   Любой новый backend обязан попадать в этот контракт.

10. **Цена / бюджет.**
    Без жёсткого `WHILLY_BUDGET_USD` агенты улетали в облако. Сделали 80%-warning
    + 100%-kill (exit 2), плюс панель cost в дэшборде (NFR-5, [#201](https://github.com/mshegolev/whilly-orchestrator/pull/201)).
