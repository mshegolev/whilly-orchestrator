<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external client → web API | untrusted credentials, CSRF tokens, session cookies cross here |
| reverse proxy → OIDC header trust | trusted-header contract; misconfiguration = auth bypass |
| external task source → orchestrator | task ids are interpolated into shell wrappers / paths / tmux targets (the ADR-001 sink class) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Spoofing | api/oidc_header_auth | mitigate | spec the fail-closed flag-gated header-trust contract (default OFF) per E17/ADR-001 |
| T-26-02 | Elevation | api/must_change_gate | mitigate | spec forced password-change gate blocking other routes until satisfied |
| T-26-03 | Tampering | core/task_id.validate_task_id | mitigate | spec the path-traversal sink-class requirement rejecting `..` and out-of-charset ids before shell/path/tmux interpolation |
| T-26-04 | Information disclosure | secrets.py / security/secret_lint | mitigate | spec secret handling + secret-lint scanning + prompt sanitization |
| T-26-05 | Repudiation | api/route_audit / auth_audit_repo | mitigate | spec route auth-audit trail recording authz outcomes |
| T-26-SC | Tampering | npm/pip/cargo installs | accept | documentation-only phase — no package installs performed |

This is a documentation-only phase (no `whilly/` code changes); mitigations are
