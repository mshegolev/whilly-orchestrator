    - `open_pr_for_task()` builds a push preflight report before the first `git push` subprocess call.
    - If `report.ok` is false, no `git push` or `gh pr create` subprocess call occurs.
    - Preflight blockers return `PRResult(ok=False, reason="rollback preflight failed: ...", failure_mode="rollback_preflight_failed", branch=<computed branch>)`.
    - Existing `git_push_failed`, `git_push_timeout`, `gh_pr_create_failed`, and `gh_pr_create_timeout` behavior remains unchanged after successful preflight.
    - Post-complete hook records `pr.open_failed` with `failure_mode="rollback_preflight_failed"` and reason from `PRResult`.
  </behavior>
  <action>
    In `whilly/sinks/github_pr.py`:
    - Import `build_preflight_report` from `whilly.rollback.service`.
    - Add an optional keyword parameter to `open_pr_for_task`: `preflight_builder: Callable[..., PreflightReport] | None = None`.
    - Default the builder to `build_preflight_report`.
    - After the worktree existence check and before `push_cmd`, call `report = builder(worktree_path, operation="push")`.
    - If `not report.ok`, return `PRResult(ok=False, branch=branch, reason="rollback preflight failed: " + "; ".join(report.blockers), failure_mode="rollback_preflight_failed")`.
    - Do not treat warnings such as missing backup point or protection unknown as blockers; use `report.ok`.
    - Preserve existing push command `git push origin HEAD:<branch> --force-with-lease`.

    Update tests:
    - `tests/test_github_pr_sink.py`: add a fake `PreflightReport` helper and prove preflight runs before push, blockers skip `_run`, and success still calls `git push` then `gh pr create`.
    - `tests/unit/test_pr_hook_failure_events.py`: add coverage that a `PRResult(failure_mode="rollback_preflight_failed")` becomes one `pr.open_failed` payload with `reason` and no `pr.opened`.
    - `tests/integration/test_post_complete_pr_hook.py`: adjust `fake_worktree` to initialize a tiny clean Git repository, or pass an injected opener/preflight path so existing PR hook tests do not depend on a plain directory that is not a Git repo.
  </action>
