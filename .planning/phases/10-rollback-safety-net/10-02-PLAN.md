  <behavior>
    - `whilly rollback create [--repo PATH] [--operation manual|push|merge|restore] [--message TEXT] [--json]` creates one rollback point and prints either human text or JSON.
    - `whilly rollback list [--repo PATH] [--branch BRANCH] [--json]` lists rollback points, with JSON as a list of `RollbackPoint.to_dict()` objects.
    - `whilly rollback preflight push|merge|restore [--repo PATH] [--target REF] [--json]` prints `PreflightReport.to_dict()` and exits `0` when ok, `1` when blockers exist, `2` on command/environment error.
    - `whilly rollback restore <tag-or-ref> [--repo PATH] [--dry-run] [--json] [--confirm PHRASE]` refuses non-TTY mutation unless `--confirm` exactly matches `restore <short-target-sha> to <branch>`.
    - `restore --dry-run --json` includes `confirmation_phrase` in output and never mutates HEAD.
  </behavior>
  <action>
    Create `whilly/cli/rollback.py` with:
    - `EXIT_OK = 0`, `EXIT_BLOCKED = 1`, `EXIT_USAGE = 2`.
    - `build_rollback_parser() -> argparse.ArgumentParser`.
    - `run_rollback_command(argv: Sequence[str]) -> int`.
    - `_print_json(payload: object) -> None` using `json.dumps(..., sort_keys=True)`.
    - Human renderers that include operation, branch, HEAD, dirty status, backup count, blockers, and warnings.

    The parser must expose exactly these subcommands and flags:
    - `create`: `--repo`, `--operation`, `--message`, `--json`.
    - `list`: `--repo`, `--branch`, `--json`.
    - `preflight`: positional `operation` choices `push`, `merge`, `restore`; `--repo`, `--target`, `--json`.
    - `restore`: positional `target`; `--repo`, `--dry-run`, `--json`, `--confirm`.

    For restore without `--dry-run`, call `restore_to_ref()` only after `--confirm` is present or after an interactive prompt returns the exact phrase. Non-TTY stdin without `--confirm` must exit `1` and print `rollback restore: confirmation required` to stderr. Do not add `--yes`.

    Add RED integration tests first in `tests/integration/test_rollback_cli.py` using real temp Git repos with local `git`:
    - `test_create_and_list_rollback_points`
    - `test_preflight_json_reports_dirty_blocker`
    - `test_restore_dry_run_outputs_confirmation_phrase`
    - `test_restore_requires_exact_confirmation`
    - `test_restore_exact_confirmation_resets_head_when_clean`
  </action>
  <acceptance_criteria>
    - `rg -n "subparsers.add_parser\\(\"create\"|subparsers.add_parser\\(\"list\"|subparsers.add_parser\\(\"preflight\"|subparsers.add_parser\\(\"restore\"|--confirm|--yes" whilly/cli/rollback.py tests/integration/test_rollback_cli.py` shows all rollback commands, `--confirm`, and no `--yes` in production CLI.
