<interfaces>
Create these public exports from `whilly/rollback/__init__.py`:
```python
from whilly.rollback.git_ops import GitClient, GitCommandResult
from whilly.rollback.models import PreflightReport, ProtectionSignal, RestoreResult, RollbackPoint, WorktreeState
from whilly.rollback.service import (
    RollbackError,
    build_preflight_report,
    confirmation_phrase,
    create_rollback_point,
    list_rollback_points,
    restore_to_ref,
)
```

Core service function contracts:
```python
def create_rollback_point(repo: Path | str = ".", *, operation: str = "manual", now: datetime | None = None) -> RollbackPoint: ...
def list_rollback_points(repo: Path | str = ".", *, branch: str | None = None) -> tuple[RollbackPoint, ...]: ...
def build_preflight_report(repo: Path | str = ".", *, operation: str, target_ref: str | None = None) -> PreflightReport: ...
def confirmation_phrase(report: PreflightReport, target_sha: str) -> str: ...
def restore_to_ref(repo: Path | str, target_ref: str, *, confirm: str = "", dry_run: bool = False) -> RestoreResult: ...
```
</interfaces>

Locked decisions:
- Rollback commands are explicit operator actions, not hidden cleanup.
- Dirty restore must stop before `git reset --hard`; confirmation is not enough to override dirty state in Phase 10.
- Do not call `git clean`, `git stash`, `git checkout .`, or any broad restore helper.
- Missing GitHub/protection data is `unknown`, not `unprotected`.
- Keep rollback outside `whilly.core` because it shells out to Git.
</context>

    - `create_rollback_point()` creates an annotated tag named `whilly/rollback/<sanitized-branch>/<YYYYmmddTHHMMSSZ>-<short-head-sha>` with `git tag -a`, never `-f`.
    - `list_rollback_points()` discovers only `refs/tags/whilly/rollback/*` tags and returns deterministic sorted `RollbackPoint` values.
    - `build_preflight_report(operation="push"|"merge"|"restore")` includes branch, HEAD SHA, upstream, dirty entries from `git status --porcelain=v1`, backup tags pointing at HEAD, and protection status.
    - Missing branch protection evidence yields `ProtectionSignal(provider="", status="unknown", reason="not requested")`, never `unprotected`.
    - `restore_to_ref()` runs preflight first, refuses dirty worktrees, requires `confirm == confirmation_phrase(report, target_sha)`, supports `dry_run=True`, and only then calls `git reset --hard <target_sha>`.
  </behavior>
  <action>
    Implement `whilly/rollback/service.py` using `GitClient` only for Git calls. Required helper behavior:
    - Resolve repo root with `git rev-parse --show-toplevel`; not-a-repo becomes a `PreflightReport` blocker and `RollbackError` for mutation commands.
    - Current branch comes from `git branch --show-current`; detached HEAD should add a blocker for merge/restore and a warning for push.
    - HEAD SHA comes from `git rev-parse HEAD`; short SHA is first 12 chars for tag names and confirmation phrase.
    - Upstream comes from `git rev-parse --abbrev-ref --symbolic-full-name @{u}` and may be `None`.
    - Dirty entries come from `git status --porcelain=v1`; include tracked and untracked entries exactly as stable porcelain lines.
    - Backup tags for current HEAD can be found with `git tag --points-at HEAD --list whilly/rollback/*`.
    - Tag validation must call `git check-ref-format refs/tags/<name>`.
    - Tag creation must call `git tag -a <name> -m <message> <head_sha>` and must not pass `-f`.
    - Confirmation phrase must be exactly `restore <short-target-sha> to <branch>`.

    Blocker policy:
    - Not a Git repository: blocker for all operations.
    - Dirty worktree: blocker for push, merge, and restore.
    - Detached HEAD: blocker for merge and restore, warning for push.
    - Confirmed protected branch: blocker. Protection `unknown` is a warning, not a blocker.
    - No backup point at current HEAD: warning, not a blocker.
