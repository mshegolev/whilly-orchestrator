<objective>
Add the repeatable, scripted entry points for the full-chain migration validation: a
`migrate-chain` Makefile target (the local operator entry point) and a `migration-chain` GitHub
Actions job (the CI entry point) that runs the same target and uploads the evidence file as an
artifact. This delivers MIG-02: chain validation is repeatable via a scripted/CI entry point, not
a one-off manual run.

Purpose: Plan 01 produced a green full-chain test. MIG-02 requires that anyone — operator or CI —
can invoke it without manual steps or operator-specific environment setup. The Makefile target is
the single canonical command; the CI job calls that same target so local and CI behavior cannot drift.
Output: A `migrate-chain` Makefile target (listed by `make help`) and a `migration-chain` CI job
that runs on ubuntu-latest and uploads migration-chain-evidence.json.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI job → Docker/testcontainers | Job boots an ephemeral Postgres; credentials are generated, not from repo secrets |
| CI job → uploaded artifact | Evidence artifact is publicly downloadable on the run; must contain no secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-04 | Information Disclosure | CI logs | mitigate | Job references no `secrets.*`; testcontainers generates ephemeral DB credentials. `make migrate-chain` asserts schema, not credentials, so no DSN is logged. |
| T-18-05 | Information Disclosure | uploaded artifact | mitigate | Artifact is the evidence JSON, which (per Plan 01 T-18-01) contains only revision string, count, and booleans — no secrets. |
| T-18-06 | Tampering | stale CI ref | mitigate | `needs: lint` + `ref: ${{ github.head_ref \|\| github.ref_name }}` ensures the job runs against the lint auto-fix commit, not a stale SHA (RESEARCH Pitfall 4). |
| T-18-SC | Tampering | pip installs | accept | `pip install -e '.[dev]'` installs only existing project deps; RESEARCH Package Legitimacy Audit rated all 4 (alembic, testcontainers, asyncpg, pytest) [OK]. No new packages added this phase. |
</threat_model>
