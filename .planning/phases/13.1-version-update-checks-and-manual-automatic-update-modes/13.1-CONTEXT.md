# Phase 13.1: Version Update Checks And Manual/Automatic Update Modes - Context

**Gathered:** 2026-05-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Deliver a safe Whilly package update mechanism before resuming WUI/TUI parity work. The phase covers
version checks, explicit manual update execution, and an opt-in automatic update policy. It does not
publish a release, change package versions, or silently hook self-update behavior into unrelated
commands.

</domain>

<decisions>
## Implementation Decisions

### Update Surface
- Use a new `whilly update` CLI namespace with `check`, `install`, and `auto` subcommands.
- `check` is non-mutating and reports installed version, latest version, and update availability.
- `install` is manual and explicit; it supports `--dry-run` plus `--installer auto|pip|pipx`.
- `auto` runs one explicit policy pass from `WHILLY_UPDATE_MODE` or `--mode off|check|install`.

### Safety Policy
- Default automatic mode is `off`.
- Automatic install is never wired into unrelated commands in this phase.
- Network and subprocess boundaries stay injectable so tests do not hit PyPI or mutate the local env.
- Unsupported installer contexts fail safely with a command/guidance message instead of guessing.

### Claude's Discretion
- Use stdlib `urllib.request` and a small numeric version comparator instead of adding runtime
  dependencies.
- Prefer pipx when a pipx environment is detected, otherwise use the current Python's pip.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `whilly/__init__.py::__version__` is the runtime installed version.
- `whilly/cli/__init__.py` lazily dispatches subcommands and keeps help text centralized.
- Existing CLI unit tests patch lazy-dispatched modules rather than spawning subprocesses.

### Established Patterns
- CLI commands return integer exit codes and write user-facing output to stdout/stderr.
- Tests use dependency injection and monkeypatching for subprocess/network boundaries.

### Integration Points
- Add `whilly.update` for service logic.
- Add `whilly/cli/update.py` for operator commands.
- Wire the `update` command into `whilly.cli.main`.

</code_context>
