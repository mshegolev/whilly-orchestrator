# Phase 13.1 Plan 01: Version Update Mechanism Summary

Implemented a classic update flow under `whilly update`:

- `whilly update check` reports installed version, latest available version, and whether an update
  is available without mutating the environment.
- `whilly update install` runs an explicit manual update using `pip` or `pipx`, with `--dry-run` for
  previewing the command.
- `whilly update auto` runs a single explicit automatic policy pass from `WHILLY_UPDATE_MODE` or
  `--mode`; default policy is `off`.

## Files Created/Modified

- `whilly/update.py` - update check, installer command, update policy, and subprocess boundary.
- `whilly/cli/update.py` - CLI commands for check/install/auto.
- `whilly/cli/__init__.py` - lazy dispatch and help text for `update`.
