Locked decisions:
- Governance risk policy must be deterministic, inspectable, pure, and not LLM-scored.
- Required category labels are exactly `migration`, `auth`, `infrastructure`, `dependencies`, `release`, and `external_pr`.
- Governance output identifies why work is high risk and what operator approval or documentation boundary applies.
- Semantic memory must be explicitly deferred unless deterministic runtime evidence is implemented and wired into planning or completion.
- Deterministic events, task history, PR evidence, and verification logs remain authoritative; semantic recall must never override them.
- Do not claim autonomous production release, default auto-merge, continuous PR review repair, full sandbox/VM isolation, or semantic long-term memory as implemented.
