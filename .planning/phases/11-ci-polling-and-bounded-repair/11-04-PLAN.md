- Consume `verification_outcome.ci_polls`, not ambiguous spec-or-result values.
- Record `ci.poll.started` from `CIPollEvidence.spec` before `ci.poll.result` and before mapped verification result events.
- Repair never calls `repo.release_task()` to retry the same failed task.
- Repair completion event wiring is in scope here; do not leave `repair.attempt.completed` as an unused primitive.

    - Insert the supplied `repair.attempt.requested` event in the same transaction.
