    - Preserve the special-case restriction that workers may only emit `human_review.required` from the `human_review.*` family.

    - The endpoint must reject non-empty dependencies with HTTP 400 and error code/text `invalid_repair_task`.

    - Add `RepairTaskResponse` containing `ok: bool` and `repair_task_id: str`.
    - `repair_task` must contain exact keys `id`, `description`, `acceptance_criteria`, `test_steps`, `prd_requirement`, `priority`, `key_files`, and `repo_target_id`; dependencies must be absent or an empty list.

    - Add `request_repair(task_id, worker_id, version, repair_task, event)`.
    - Send worker id, original task id/version, repair task payload, and event payload to `/tasks/{task_id}/repair`.
    - Return the server `repair_task_id` and raise existing HTTP client errors on 4xx/5xx.
    - Keep `record_event()` compatible with `ci.*` and `repair.*` event payloads.
