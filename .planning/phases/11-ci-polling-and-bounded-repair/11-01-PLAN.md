<interfaces>
Existing verification runner contract:
```python
@dataclass(frozen=True)
class VerificationCommandSpec:
    name: str
    command: str
    required: bool = True
    source: str = CLI_VERIFICATION_SOURCE

async def run_verification_commands(
    commands: list[VerificationCommandLike] | tuple[VerificationCommandLike, ...],
    *,
    cwd: str | Path,
    timeout_s: float = DEFAULT_TIMEOUT_S,
    env_allowlist: tuple[str, ...] = (),
    output_limit: int = DEFAULT_OUTPUT_LIMIT,
) -> VerificationRunOutcome: ...
```

Existing event builder shape:
```python
@dataclass(frozen=True, slots=True)
class PipelineTaskEvent:
    task_id: str
    event_type: str
    payload: dict[str, Any]
    detail: Mapping[str, Any] | None = None
```
</interfaces>

Locked decisions:
- CI polling is explicit configured verification evidence, not a background daemon.
- Missing, unavailable, unauthenticated, timed-out, or unknown CI data is explicit and not success.
- `source="ci"` verification must never be passed to `asyncio.create_subprocess_shell`.
- Reuse the existing `gh`/PR feedback shape where useful, but do not convert PR feedback polling into an always-on loop.
</context>
