<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.whilly_pause` file → loop | operator-authored pause reason crosses into log/audit payload |
| plan JSON readiness → dispatch decision | on-disk verdict gates whether autonomous work runs |
| loop → dispatch runner | autonomous work is launched only through the gated runner |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-06 | Elevation of Privilege | dispatch without gate | mitigate | dispatch is structurally unreachable unless `--dispatch` set AND unpaused AND readiness satisfied (or explicit `--allow-unready-run`); routed through the injected Phase-17-gated runner |
| T-20-07 | Information Disclosure | pause reason / readiness payload | mitigate | watch.paused/watch.block payloads carry only reason/verdict/missing_context/issue_key — no token, no DSN |
| T-20-08 | Spoofing | circular-import bypass of the gated runner | mitigate | loop never imports `whilly.cli.jira`; dispatch hook is injected (verified by grep gate in `<done>`) |
| T-20-09 | Tampering | forged readiness verdict in plan JSON | accept | plan JSON is operator-owned local state; same trust model as `_run_intake`; out of scope to harden file ownership here |
</threat_model>
