## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps, status/action adjacency, selected-row outline offset |
| sm | 8px | Mobile row gaps, stacked-cell padding, action-button gaps |
| md | 16px | Mobile shell inset, row group separation, topbar wrapping rhythm |
| lg | 24px | Section spacing when a mobile surface needs separation |
| xl | 32px | Reserved for wide layout gaps only |
| 2xl | 48px | Reserved for page-level breaks; do not introduce in Phase 6 |
| 3xl | 64px | Reserved for page-level breaks; do not introduce in Phase 6 |

Exceptions: mobile touch targets use `44px` minimum width and height for review-action buttons and any
new icon-sized action control. `44px` stays on the 4px grid. No other non-grid spacing is allowed.

Phase 6 must preserve existing desktop table cell padding and apply mobile-only spacing inside
`@media (max-width: 900px)`. Mobile stacked rows use `8px` row gaps, `4px 8px` cell padding, and
`8px` internal label/value gaps.

## Visual Hierarchy

The active operator surface remains the primary anchor. On mobile, each table row becomes the
primary scan unit:

| Surface | Mobile primary line | Secondary details | Action zone |
|---------|---------------------|-------------------|-------------|
| Compliance | `Task` plus `Reason` | `Plan`, `Stage`, `Reviewer` | `Actions` full-width at row bottom |
| Plans/Tasks | `Task` plus `Status` and `Priority` | `Plan`, `Worker`, `Review`, `Updated` | none |
| Workers | `Worker` plus `Status` | `Hostname`, `Owner`, `Last heartbeat` | none |
| Events | `Id` plus `Type` | `Task`, `Plan`, `At` | none |

The queue-health metric strip remains above surfaces and keeps its existing two-column mobile grid.
Do not introduce nested cards. The row itself may have a single `1px` border and `4px` radius, but
cells inside that row must remain unframed.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 | 1.45 |
| Label | 12px | 600 | 1.2 |
| Heading | 15px | 600 | 1.2 |
| Display | 18px | 600 | 1.2 |

Use only weights `400` and `600` for newly touched mobile row UI. Mobile cell labels use the Label
role; row values use Body. Do not scale font size with viewport width. Long task IDs, worker IDs,
hostnames, event types, and timestamps must wrap with `overflow-wrap: anywhere` instead of forcing
horizontal scroll.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#f7f8fa`; dark `#111315` | Page background and operator-console negative space |
| Secondary (30%) | Light `#ffffff` / `#f1f3f6`; dark `#171a1d` / `#202428` | Tables, stacked mobile rows, metric strip, tabs, controls |
| Accent (10%) | `#0f766e` | Selected review row outline, focus outline, subtle row hover/active background |
| Destructive | `#b42318` | Failed status, rejected review state, reject action border/status only |

Accent reserved for: selected actionable review row outline, keyboard/touch focus outline, and subtle
row hover/active background. Do not use accent for all mobile row borders, generic labels, task IDs,
or every action button.

Existing semantic status colors remain data-state colors only: ok `#16703c`, warning `#a15c07`,
info `#0969da`, destructive `#b42318`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Approve review` as the accessible action name; visible compact label remains `A` until Phase 7 |
| Empty state heading | none - empty states remain a single row inside the affected table |
| Empty state body | Tasks: `No tasks in queue.`; Workers: `No workers connected yet.`; Review gaps: `No human review or verification gaps.`; Events: `No events recorded.` |
| Error state | `Control plane unavailable` plus the error detail and `Retry` |
| Destructive confirmation | `Reject selected review gap`: prompt `Review comment`; `Request changes`: prompt `Requested changes`; cancel aborts |

Phase 6 adds `aria-label` values for mobile review buttons: `Approve review for {task_id}`,
`Reject review for {task_id}`, and `Request changes for {task_id}`. Do not change visible `A`, `X`,
or `C` labels in this phase; clearer visible labels and stronger recovery affordances are Phase 7.

## Mobile Row Layout Contract

Breakpoint: all mobile row layout rules apply at `max-width: 900px`, matching the existing WUI
responsive block. Above 900px, desktop table behavior, headers, column order, and horizontal overflow
fallback remain unchanged.

### Shared Rules

| Rule | Contract |
|------|----------|
| Header source | Visible mobile labels must come from the Phase 5 `table_columns.*` metadata, not hard-coded duplicate strings. |
| Row structure | Add per-cell mobile labels with `data-mobile-label="{{ column.label_for('wui') }}"`; mobile CSS renders those labels before values. |
| Desktop preservation | Do not remove `<thead>`, `role="grid"`, existing table ids, row ids, HTMX attributes, or `data-filter-text`. |
| Mobile row display | Each body row becomes a full-width stacked row with `1px` line border, `4px` radius, `8px` bottom gap, and no horizontal scroll requirement for critical controls. |
| Cell layout | Each non-action cell displays as a two-column label/value grid: label `minmax(84px, 32%)`, value `minmax(0, 1fr)`. Action cells span the full row. |
| Wrapping | Override `white-space: nowrap` for mobile row cells. Values must wrap cleanly and must not overlap adjacent controls. |
| Empty rows | Empty-state rows span the row width and keep existing copy. They do not need label/value presentation. |
| Fragments | `_tasks_table.html` and `_workers_table.html` fragments must include the same mobile-label attributes as the full page render. |

### Compliance Review Rows

Review rows are the only Phase 6 rows with critical actions.

| Element | Contract |
|---------|----------|
| Details | Show `Task`, `Plan`, `Reason`, `Stage`, and `Reviewer` as stacked labeled fields in Phase 5 order. |
| Action placement | The `Actions` field is a full-width row section after details, never inline beside `Reason` or `Reviewer`. |
| Action sizing | `A`, `X`, and `C` buttons use `min-width: 44px`, `min-height: 44px`, `padding: 8px`, and `8px` gap. |
| Action grouping | Action buttons remain in one wrapping row. If width is too narrow, buttons wrap without shrinking below `44px`. |
| Read-only gaps | Non-actionable rows show `read-only` in the same full-width action zone, with no disabled fake buttons. |
| Selection | Tapping an actionable review row selects it; tapping a review-action button submits that row's action. Selection outline remains accent `#0f766e`. |

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Filter | Global filter behavior and `data-filter-text` coverage remain unchanged across full-page and fragment refreshes. Hidden rows must stay hidden in mobile layout. |
| Review selection | `j/k` selection remains Compliance-only. Mobile tap selection applies only to actionable review rows. |
| Review actions | `a/x/c` hotkeys and `A/X/C` buttons continue mapping to `approved`, `rejected`, and `changes_requested` through the shared review-decision path. |
| Touch behavior | Critical review buttons must be operable without precise horizontal scrolling. No action button may overlap text at widths below 900px. |
| Refresh | HTMX polling, SSE swaps, and manual refresh continue preserving active surface, filter text, selected review row, and allowed input focus. |
| Desktop behavior | Above 900px, existing table headers, cell order, nowrap behavior, and `.table-wrapper { overflow-x: auto; }` remain unchanged except for harmless data attributes. |
| Accessibility | Keep row ids, table ids, and `aria-selected`. Add button `aria-label`s for review actions if visible labels remain compact. Focus indicators must be visible in light and dark schemes. |
