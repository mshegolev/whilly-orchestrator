<objective>
Wire the Plan 01 core functions into an end-to-end single-spec review and a
`--slug` CLI entry (DETECT-01). This plan adds:
1. `review_spec(slug, reviewer, ...)` — the pipeline: load the spec's `spec.md`,
   load the mapped module sources (via `resolve_modules_for_slug`), build the
   prompt (`build_review_prompt`), call the injected `reviewer` callable, and
   parse the result (`parse_findings`). Returns `list[dict]` findings.
2. A default reviewer that shells to the Claude CLI (`claude -p --output-format
   json`, honoring `CLAUDE_BIN`) — kept thin per CONTEXT (no full retry stack).
   The Claude `--output-format json` envelope is unwrapped to the inner `result`
   text before parsing.
3. `main()` — argparse `--slug <capability>`, runs `review_spec` with the default
   reviewer, prints findings JSON to stdout, exits 0 always.

Purpose: deliver the operator-facing single-spec checker on top of the
test-locked core. Reviewer is dependency-injected so the pipeline is testable
with zero network/CLI; live-CLI test skips when `claude` is absent (repo
convention).
Output: `scripts/semantic_drift_check.py` (pipeline + reviewer + CLI) and
appended tests in `tests/test_semantic_drift_check.py`.
</objective>
