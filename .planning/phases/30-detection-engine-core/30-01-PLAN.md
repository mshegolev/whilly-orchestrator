<objective>
Build the pure, model-free core of the single-spec semantic drift engine in a
standalone tooling script under `scripts/` (NOT under `whilly/` — per CONTEXT
decision, anything in `whilly/` carries a coverage-matrix + opsx-spec obligation;
this tool is exempt like `scripts/audit-coverage-matrix.py`).

This plan delivers three deterministic, fully testable functions and their tests:
1. `resolve_modules_for_slug(slug)` — derive the spec→module review set live from
   `openspec/COVERAGE-MATRIX.md`, reusing the matrix-parsing approach already in
   `scripts/audit-coverage-matrix.py` (DETECT-04).
2. `build_review_prompt(...)` — a PURE function that assembles the deterministic
   review prompt embedding spec text + mapped module sources and instructing the
   model to back every finding with `file:line` and triage code-bug vs
   spec-overstatement (DETECT-02, DETECT-03 prompt contract).
3. `parse_findings(text)` + `validate_finding(d)` — robust JSON extraction
   (fence-strip + json_repair fallback, mirroring `whilly/prd_generator.py`) and
   per-finding schema validation (DETECT-02, DETECT-03 schema).

Purpose: These are the contracts Plan 02 wires into a CLI + live reviewer. They
must be correct and test-locked before any model or subprocess is involved.
Output: `scripts/semantic_drift_check.py` (core functions) and
`tests/test_semantic_drift_check.py` (unit tests, no network/CLI).
</objective>
