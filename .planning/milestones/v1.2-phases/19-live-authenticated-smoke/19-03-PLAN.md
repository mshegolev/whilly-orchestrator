<objective>
Create the new `whilly gitlab` CLI group with a `smoke --repo-url URL` action,
register it in `whilly/cli/__init__.py` (lazy dispatch + `_HELP_TEXT`), and add
a thin injectable `_gitlab_get` urllib client. The smoke does a
token-authenticated API ping (`/api/v4/user`), resolves the project from the
repo URL (`/api/v4/projects/{encoded path}`), and validates repo-hint
extraction — all read-only — accumulating results into the shared `SmokeReport`
and writing a redacted JSON report.

Purpose: LIVE-02 (authenticated GitLab smoke: link-refresh + repo-hints) and
LIVE-03 (persisted report). Resolves open questions: urllib-based `_gitlab_get`
(not glab subprocess); token precedence GITLAB_TOKEN → GITLAB_API_TOKEN →
WHILLY_GITLAB_API_TOKEN; link-refresh = parse repo-hint from `--repo-url` and
verify project accessibility via the authenticated API.
Output: `whilly/cli/gitlab.py`, `whilly/cli/__init__.py` (registration),
`tests/unit/cli/test_gitlab_smoke.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator `--repo-url` → GitLab REST path + report filename | Untrusted input flows into project-path construction |
| GitLab server JSON response → report payload / path components | Server response is untrusted; must not be used as a traversal path |
| GitLab token (env/glab) → report payload / stdout | Token must never be persisted or printed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-06 | Tampering | `--repo-url` path component into `/api/v4/projects/{path}` and report filename | mitigate | `_resolve_project_path` strips scheme/host/`.git` and normalizes (`_normalize_repo_path` strips `../`) before URL-encoding; report filename `kind` is a fixed literal |
| T-19-07 | Tampering | malicious GitLab response used as a path/filename component | mitigate | response fields used only for boolean match comparison and counts, never as a filesystem path; report filename derived from fixed kind + generated timestamp |
| T-19-08 | Information Disclosure | report payload + stdout in `_run_gitlab_smoke` | mitigate | only `_redact_url` host, repo path, booleans, counts written; token never serialized; Task 3 asserts no token/DSN in report |
| T-19-09 | Information Disclosure | raw urllib exception surfaced to operator | mitigate | `_gitlab_get` converts HTTPError/URLError to RuntimeError; per-check catch converts to hint; Task 3 asserts no "Traceback" |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No new packages this phase (RESEARCH audit: none) |
</threat_model>
