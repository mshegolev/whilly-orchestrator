<objective>
Create the shared smoke-report foundation consumed by both `whilly jira smoke`
and `whilly gitlab smoke`: a `SmokeReport` accumulator dataclass, a
`write_smoke_report()` helper that writes a timestamped secret-free JSON file
under `whilly_logs/smoke/`, a `_redact_url()` helper, and the three shared exit
code constants. This is the interface contract the Wave 2 command plans build
against.

Purpose: LIVE-03 — every smoke run must leave persisted, redacted audit
evidence. Centralizing the report writer here keeps redaction (a locked
security decision) in one auditable place and prevents per-command drift.
Output: `whilly/cli/smoke.py`, `tests/unit/cli/__init__.py`, `tests/unit/cli/test_smoke.py`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| in-memory check results → report file on disk | Redaction must happen before any secret-bearing value crosses to the persisted file |
| operator-supplied URLs (server/repo) → report payload | URLs may embed `user:pass@` credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Information Disclosure | `write_smoke_report` report payload | mitigate | `_redact_url` strips userinfo; only host/identifiers/booleans/counts serialized; Task 2 asserts no token/DSN substrings in output |
| T-19-02 | Tampering | report filename built from `kind` + timestamp | mitigate | `kind` is a fixed internal literal ("jira"/"gitlab"), not operator input; timestamp is generated, not supplied |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this phase (RESEARCH Package Legitimacy Audit: none) |
</threat_model>
