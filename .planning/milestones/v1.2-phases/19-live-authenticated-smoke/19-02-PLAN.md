<objective>
Add `whilly jira smoke --issue KEY` as a new ACTION in the existing
`whilly jira` subparser. It reuses the poll-cycle code path
(`collect_jira_work_snapshot`) read-only, accumulates per-check results
(auth/whoami, issue fetch, comments, changelog, remote links, classify) into a
`SmokeReport`, prints a human-readable summary, writes a redacted JSON report,
and returns exit 0/1/2.

Purpose: LIVE-01 (authenticated Jira smoke with classify/history/comments/links)
and LIVE-03 (persisted report). The Jira check path already exists in production
code; this plan wires it together in read-only smoke mode with actionable
failure hints instead of tracebacks.
Output: `whilly/cli/jira.py` (smoke action), `tests/unit/cli/test_jira_smoke.py`.
</objective>
