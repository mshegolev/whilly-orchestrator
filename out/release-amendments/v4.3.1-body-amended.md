## Highlights

**Hotfix + 4th agentic CLI in one drop.** Production image now ships
with **four** coding agents preinstalled (claude-code, gemini-cli,
opencode, **codex**), all running on Node 22 LTS. The previous v4.3.0
shipped Node 18 (Debian bookworm default), which silently broke
`gemini --version` with `SyntaxError: Invalid regular expression flags`
because gemini-cli requires Node ≥20. v4.3.1 fixes that and adds
OpenAI's Codex CLI (gpt-5.x family) as the 4th option through the
existing `docker/cli_adapter.py` dispatcher.

## Fixed

- **`@google/gemini-cli` no longer crashes in the production image.**
  Replaced Debian's stock `apt install nodejs` (Node 18.20 on bookworm)
  with NodeSource's `setup_22.x` (Node 22 LTS, includes npm). Affects
  both `Dockerfile` and `Dockerfile.demo`. Build-time sanity now also
  invokes `--version` on each CLI, so a future Node downgrade fails the
  build instead of breaking the runtime.

## Added

- **Codex CLI (`--cli codex`).** OpenAI's official Codex CLI
  (`@openai/codex`, v0.128+) is now part of the production image and
  exposes the same surface as the other three: sub-agents, skills, MCP,
  plugins, AGENTS.md, hooks, sandbox modes. `docker/cli_adapter.py`
  gained a `run_codex(prompt, model)` runner that:
  - drives `codex exec --json --skip-git-repo-check
    --dangerously-bypass-approvals-and-sandbox --ephemeral` (sandbox
    + approvals OFF — container-level isolation is sufficient)
  - reads the final agent message from the temp file passed via
    `-o <file>` (with stream-event fallback if the file is empty)
  - sums `input_tokens`, `output_tokens`, and
    `reasoning_output_tokens` across all `turn.completed` events for
    Whilly's accounting
  - classifies 401/403 stderr signatures as **permanent** failures
    (exit 2) so whilly-worker doesn't retry-loop on bad credentials
- **`openai` provider in `llm_resource_picker`.** Tier→model mapping:
  TINY/SMALL → `gpt-5.4-mini` (fast/cheap), MEDIUM/LARGE → `gpt-5.4`
  (flagship for API-key auth; `gpt-5.5` is locked to ChatGPT Pro/Plus
  subscriptions and therefore not used in the default map). Override
  via `LLM_MODEL`.
- **`workshop-demo.sh --cli codex`.** Requires `OPENAI_API_KEY`. Sets
  `CODEX_HOME=/home/whilly/.codex` so config + auth cache + skills
  + plugins + MCP can be volume-mounted read-only from the host.
- **10 new `TestCodexAdapter` unit cases** covering argv shape,
  single + multi-turn usage aggregation, last-message file fallback to
  stream events, and auth/general/timeout/missing-binary failure modes.

## Changed

- The production image now contains four agentic CLIs (was three):
  claude / gemini / opencode / **codex**. Image size grew by ~150 MB
  (codex bundle), now ~1.6 GB.
- `cli_adapter._RUNNERS` keys: `{claude-code, opencode, gemini, codex}`.
- DEMO.md updated: `--cli codex` example, comparison table with codex
  row, codex `~/.codex/` skills/plugins/MCP volume-mount example,
  cgroup-aware tier table now lists OpenAI (codex) models.
