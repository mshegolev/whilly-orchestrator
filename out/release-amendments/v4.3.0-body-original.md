## Highlights

**Pull-and-go agentic workflow.** Production Docker image `mshegolev/whilly:4.3.0` теперь включает три полноценных кодинг-агента — Claude Code, Gemini CLI, OpenCode — со своими sub-agents, skills, MCP-серверами и file-tools. Whilly worker внутри контейнера может зайти в любой из них через единый adapter без правки кода.

## What's new

### Agentic CLI mode (`--cli claude-code|opencode|gemini`)

`docker/cli_adapter.py` — Python adapter, который whilly-worker зовёт как `$CLAUDE_BIN`. Транслирует whilly-style argv в native argv каждого CLI и парсит native output обратно в whilly envelope.

| CLI            | Provider lock      | Free path                       |
|----------------|--------------------|---------------------------------|
| **claude-code**| Anthropic only     | нет (paid Anthropic API)        |
| **gemini-cli** | Google Gemini only | 1500 req/day Gemini 2.0 Flash   |
| **opencode**   | любой через models.dev | OpenRouter `:free` модели    |

Ключевая разница от raw LLM shim'а (v4.2.0): agentic CLIs тащат свои **sub-agents**, **skills** (`~/.claude/skills`), **MCP-серверы** и **file-tools** — это полноценные кодинг-агенты, не голые модели.

### Sub-agents и skills

Все три CLI читают конфигурацию из стандартизированных директорий: `~/.claude/CLAUDE.md` (глобальный системный промпт), `~/.claude/agents/<name>.md` (sub-agent определение), `~/.claude/skills/<name>.md` (переиспользуемая markdown-инструкция). В Docker-контейнере воркера эти файлы пробрасываются через volume mount.

## Quick demo

```bash
git clone https://github.com/mshegolev/whilly-orchestrator
cd whilly-orchestrator

# Architectural demo (stub Claude — no API keys needed):
./workshop-demo.sh --workers 2 --cli stub

# Real agentic workflow with Anthropic Claude Code:
export ANTHROPIC_API_KEY=YOUR_KEY
./workshop-demo.sh --workers 2 --cli claude-code

# Real agentic workflow with free OpenRouter via OpenCode:
export OPENROUTER_API_KEY=YOUR_KEY
./workshop-demo.sh --workers 2 --cli opencode

# Real agentic workflow with free Gemini:
export GEMINI_API_KEY=YOUR_KEY
./workshop-demo.sh --workers 2 --cli gemini
```
